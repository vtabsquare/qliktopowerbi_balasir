12. Power Query conversion rules

The AI must convert Qlik ETL into deterministic Power Query steps.

Source loads
LOAD ...
FROM [file.csv]
(txt, utf8, embedded labels, delimiter is ',', msq);

Convert to:

Csv.Document(
    File.Contents(...),
    [
        Delimiter = ",",
        Encoding = 65001,
        QuoteStyle = QuoteStyle.Csv
    ]
)

Honor Qlik source metadata:

Encoding
Codepage
Delimiter
Header mode
Quote handling
Sheet name
XML or JSON structure
File path variables

Do not hard-code UTF-8 for all sources.

Resident loads
LOAD ...
RESIDENT SourceTable;

Convert to a reference to the upstream M query or M step.

Preceding loads

Compile in bottom-up execution order while preserving Qlik semantics.

Concatenate
CONCATENATE(FactSales)
LOAD * FROM ...;

Convert to:

Table.Combine({Table1, Table2})

Before combining:

Harmonize schemas
Add missing columns
Preserve common field names
Validate datatypes
Respect NOCONCATENATE
Joins
LEFT JOIN (FactSales)
LOAD ...

Convert to:

Table.NestedJoin(...)

followed by controlled expansion.

Before joining:

Determine natural Qlik join keys
Validate datatype compatibility
Detect duplicate-key risk
Detect row multiplication
Detect name collisions
Prevent duplicate expansion names
Mapping and ApplyMap
RegionMap:
MAPPING LOAD RegionID, RegionName ...;

ApplyMap(
    'RegionMap',
    RegionID,
    'Unknown Region'
)

Generate either:

A valid named lookup query
An inline lookup transformation
A record-based lookup for small mappings
A merge-based lookup for larger mappings

The generated query must never reference a missing name such as:

#"RegionMap"

unless that query exists in the PBIP project.

Preserve default fallback behavior exactly.

Calculated fields

Convert row-level Qlik calculations to Table.AddColumn.

Use null-safe logic where Qlik and M differ.
