======================================================================
PRIMARY OBJECTIVE
======================================================================

Enhance the application so that:

1. Every expression identified through QVW UI analysis is captured in a centralized expression inventory.
2. Each QlikView expression is translated into an appropriate Power BI implementation.
3. Converted expressions are written into the generated PBIP semantic model.
4. Generated report visuals reference the converted measures, columns, parameters and fields.
5. The application displays a complete Power BI Data Model view before export.
6. Users can add, edit, delete, activate, deactivate and validate relationships.
7. Relationship changes made in the UI are persisted into the final PBIP package.
8. Unsupported or partially converted expressions are never silently ignored.
9. The final PBIP package opens successfully and contains the reviewed semantic model, expressions, relationships and report visuals.

======================================================================
NON-NEGOTIABLE ENGINEERING RULES
======================================================================

Do not generate placeholder files, dummy functions, mocked conversion results or sample-only logic.

Do not hardcode object IDs, table names, field names, expression names, sheet names or test-package values.

Do not overwrite existing functionality that already works.
